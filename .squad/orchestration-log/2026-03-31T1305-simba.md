## Key Decisions

1. **Translation library:** `google-translate-api-x` provides free translation without API key.
2. **Module boundary:** All translation logic isolated in `translator.ts`; pipeline orchestrates the call.
3. **HTML preservation:** Large HTML chunked at block boundaries (≤4000 chars) to avoid API limits; tags preserved via API's `autoCorrect` mode.
4. **Resilience:** Translation failure gracefully returns original content with warning; never crashes pipeline.
5. **Filename convention:** Language code appended to filenames (e.g., `example-com-he.pdf`), composable with `--compress` suffix.
6. **originalTitle field:** Stores pre-translation title so Rafiki's PDF generator can reference it for bilingual display.

## Integration Points

- Pipeline calls `translateContent()` after extraction, before PDF generation.
- In crawl mode, translation progress logged per page.
- Filename generation updated to incorporate language suffix when translate flag is used.
- Translator module owns error handling; never throws.
